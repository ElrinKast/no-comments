# Kolink call media notes

Дата фикса: 2026-06-29.

Рабочая схема звонков держится на этих правилах:

1. Участник, который уже был в звонке, всегда создает WebRTC offer для нового участника.
2. Новый участник создает peer как отвечающая сторона и ждет offer.
3. Нельзя выбирать инициатора по случайному `socket.id`: это ломало переговоры и давало черные плитки `VOICE`.
4. `onnegotiationneeded` должен быть назначен до ручного запуска переговоров, а offer нужно ставить в очередь через `queuePeerNegotiation`.
5. Отвечающая сторона не должна заранее добавлять свои audio/video transceiver до получения offer. После `setRemoteDescription(offer)` она добавляет свои треки в уже существующие линии и только потом создает answer.
6. Удаленный звук проигрывается отдельным `<audio autoplay playsinline>`, а `<video>` используется только для картинки. Это стабильнее для телефонов и браузеров.
7. Состояние камеры отправляется на сервер через `call:join` и `call:media`, а в presence есть `cameraOn`. Так выключенная камера выглядит как `VOICE`, а не как сломанный черный экран.
8. Демонстрация экрана заменяет текущий video track через `replaceTrack`, затем инициатор запускает renegotiation для peer, где это нужно.

Коммиты, которые привели звонки в рабочее состояние:

- `885c8a2 Fix call media negotiation`
- `457336a Fix answerer media transceivers`

Как проверять после будущих изменений:

1. Открыть два клиента `https://app.kolink.ru`.
2. Зайти в звонок с обоих клиентов.
3. Проверить, что оба видят и слышат друг друга.
4. С ПК включить `screen` и проверить, что второй клиент видит демонстрацию.
5. Проверить `cam/off` на телефоне и ПК.
